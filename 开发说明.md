
## commit 消息格式

<type>(<scope>): <subject>

比如：
gti commit -m 'feat: add new ..'

## commit type 说明

|Type	    |说明       |适用场景|
|-----------|----------|--------|
|feat	    |新功能     |增加了新逻辑、新页面或新组件|
|fix	    |修补 Bug	|修复了任何程序错误|
|docs	    |文档变更	|修改了 README, JSDoc 或其他文档|
|style	    |格式变动	|不影响代码逻辑的改动（空格、格式化、分号等）|
|refactor	|重构	    |既不是修复 Bug 也不是新增功能的代码更改|
|perf	    |性能优化	|提高运行速度、优化内存占用|
|test	    |测试	    |添加缺失的测试或修正现有的测试|
|build	    |构建系统	|修改 npm 依赖、打包配置 (gulp, webpack)|
|ci	        |持续集成	|修改 GitHub Actions, Travis, Docker 配置文件|
|chore	    |杂务	    |修改 .gitignore, 这里的 init, 或非源码变动|
|revert	    |回退	    |撤销之前的 commit|
